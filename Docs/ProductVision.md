# Ozen: Product Vision & Philosophy

## The Problem
There is a cognitive gap between "Action" and "Knowledge." Currently, when a user needs information or an AI action, they must:
1. Leave their current context (browser tab, IDE, Word doc).
2. Open a browser or a specific app.
3. Type the query.
4. Copy-paste result back.

**Ozen kills this gap.**

## The Vision
Ozen is a **near-invisible layer** that sits on top of the OS. It doesn't want you to come to it; it comes to you when you summon it via `@ozen`.

### Core Experience Pillars
12. **Contextual Presence**: Triggered at the cursor position. No jumping to a side window.
3. **The Orb (Status Indicator)**: A small, always-on-top floating icon that shows Ozen's state (Active/Inactive) and allows for quick toggling.
4. **Hybrid Intelligence**: Combines the reasoning of local LLMs (Ollama) with the up-to-date reality of the live Web.
5. **Fluidity**: Transitions seamlessly from a tiny prompt-bar to a full-fledged "Power Interface."

## Primary Use Cases
- **The Magic Prompt**: Typing `@ozen write a regex for email` in VS Code pops up the code immediately.
- **The Search Companion**: `@ozen penguins` shows an AI summary *plus* a live Google/DuckDuckGo result in a floating frame.
- **The Deep Dive**: Clicking "Extend" on a panel to open the Main App for complex workflows, browser management, and agent configuration.
